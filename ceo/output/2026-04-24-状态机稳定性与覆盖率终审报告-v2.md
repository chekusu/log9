## 状态机稳定性审查结论

### `apps/api`

已关闭的关键风险：

- `ingest` 从“返回成功但静默丢数不可观测”收敛为明确的 `202 deferred persistence` 语义。
- `waitUntil(...)` 后台持久化失败现在会记录错误并保留 rejection，可观测性成立。
- `query` 与 `db9 gateway` 已共享只读 SQL 门禁，覆盖单语句、只读入口、���险关键字、锁定读等边界。

当前评价：

- 稳定性显著提升，关键状态迁移已有测试固定。
- 剩余风险在于 SQL 校验仍是规则式防护，不是完整 SQL 解析器。

### `packages/core`

已验证的关键行为：

- batch flush、timer flush、manual flush、空 flush 全覆盖。
- 事件与 span 的缓冲、发送与错误吞没路径已被测试固定。
- breadcrumb ring buffer 限长行为已覆盖。

当前评价：

- 行为稳定，测试完整。
- 残余风险是传输语义仍为 best-effort；发送失败不会重试，也没有 dead-letter。

### `packages/sdk-cloudflare`

已关闭的关键风险：

- throw 场景的重复异常事件与重复 flush 风险已修复。
- `withErrorCapture()` 的嵌套异常捕获已做到同一错误对象幂等上报。
- `2xx / 4xx / 5xx / throw` 的事件、span、`flush`、`waitUntil` 调用次数均已被精确断言。

当前评价：

- 异常路径语义已清晰且稳定。
- 残余风险是同步 `flush()` 若直接抛错，仍会影响请求成功路径，这一行为目前是保留语义。
