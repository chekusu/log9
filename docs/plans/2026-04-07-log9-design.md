# log9.ai 设计文档

> AI-native 集中式可观测性平台：SDK 收日志 → db9 存储 → Agent 自动修 → 人类用自然语言查

## 定位

log9.ai 是一个通用的日志集中平台，面向独立开发者同时运营多个 SaaS 产品的场景。核心差异化：不只收日志，还通过 agentic loop 自动发现问题、修复代码、补全测试、部署上线。

## 架构总览

```
  所有来源                  log9.ai                           存储
  ────────                 ────────                          ──────
  @log9/sdk ──┐
  Twilio ─────┤     ┌─────────────────────────────┐
  Agora ──────┼────▶│  Log Worker (Cloudflare)     │     ┌─────────┐
  Stripe ─────┤     │  POST /ingest/:project/:type │────▶│ db9.ai  │
  Custom ─────┘     │  标准化 + 验签 + 写入 db9     │     │(Postgres)│
                    └─────────────────────────────┘     └────┬────┘
                                                             │
                    ┌─────────────────────────────┐          │
  人类查询 ────────▶│  Query Worker (Cloudflare)   │◀────────┘
  "最近有什么错误？" │  NL → SQL → Dynamic Worker   │
                    └─────────────────────────────┘
                                                             │
                    ┌─────────────────────────────┐          │
                    │  Log9 Agent (wanman runtime) │◀────────┘
                    │  持续分析 → 发现 → 修复 → 部署 │
                    └─────────────────────────────┘

  接入产品: tuwa.ai / saifuri / menkr / kimeeru / m0rphic / ...
```

**核心原则：所有日志统一进 Log Worker，SDK 不直连 db9。**

## 第一部分：@log9/sdk

### 分层架构（参考 Sentry JS SDK）

```
@log9/core             ← 基础层：event schema、breadcrumbs、批量发送
@log9/cloudflare       ← Cloudflare Workers 自动捕获（最先做）
@log9/node             ← Node.js 运行时（wanman sandbox、本地服务）
@log9/browser          ← 浏览器端（后期）
```

SDK 只负责采集 + POST JSON 到 Log Worker，不直连 db9。

### 自动捕获能力

| 能力 | 说明 |
|------|------|
| Error Capture | 自动捕获 uncaught exceptions + unhandled rejections |
| Request Logging | 拦截每个 fetch，记录 method、path、status、latency |
| Breadcrumbs | 自动串联一次请求内的所有事件 |
| Context | 自动附加 env 信息（Worker name、region、request ID） |
| Performance | 记录 CPU time、wall time、sub-request 数量 |

### db9 表结构

```sql
-- 事件表（每条日志一行）
CREATE TABLE events (
  id          TEXT PRIMARY KEY,
  project     TEXT NOT NULL,        -- 'tuwa', 'saifuri', etc.
  level       TEXT NOT NULL,        -- 'error', 'warn', 'info', 'debug'
  message     TEXT NOT NULL,
  timestamp   TIMESTAMPTZ NOT NULL,
  trace_id    TEXT,                 -- 关联同一次请求的所有事件
  tags        JSONB,               -- { service, worker, region, ... }
  extra       JSONB,               -- 业务自定义数据
  breadcrumbs JSONB,               -- 时间线面包屑
  stack_trace TEXT                  -- 错误堆栈
);

-- 性能表（每次请求一行）
CREATE TABLE spans (
  id          TEXT PRIMARY KEY,
  project     TEXT NOT NULL,
  trace_id    TEXT NOT NULL,
  name        TEXT NOT NULL,        -- 'fetch /api/call', 'gemini.connect'
  started_at  TIMESTAMPTZ NOT NULL,
  duration    FLOAT NOT NULL,       -- ms
  status      INT,                  -- HTTP status or 0
  tags        JSONB
);

-- 索引
CREATE INDEX idx_events_project_ts ON events (project, timestamp DESC);
CREATE INDEX idx_events_level ON events (level) WHERE level IN ('error', 'warn');
CREATE INDEX idx_events_trace ON events (trace_id) WHERE trace_id IS NOT NULL;
CREATE INDEX idx_spans_project_ts ON spans (project, started_at DESC);
CREATE INDEX idx_spans_trace ON spans (trace_id);
```

### 第三方日志接入

SDK 管不到的第三方平台，通过 Webhook Ingestion 端统一接入：

| 来源 | 方式 |
|------|------|
| 自己的代码 | **@log9/sdk** Push |
| Twilio | Status Callback → `POST /ingest/:project/twilio` |
| Agora | Event Notification → `POST /ingest/:project/agora` |
| Stripe | Webhook → `POST /ingest/:project/stripe` |
| GitHub | Webhook → `POST /ingest/:project/github` |
| Gemini/OpenAI | SDK 内拦截：wrap fetch 自动记录 LLM 请求/响应 |
| 任意服务 | `POST /ingest/:project/custom`（通用 JSON） |

### Webhook Ingestion

```
第三方服务                         log9.ai API Worker
┌──────────┐                      ┌──────────────────────────────┐
│ Twilio   │──callback───────────▶│ POST /ingest/:project/twilio │
│ Agora    │──notification───────▶│ POST /ingest/:project/agora  │
│ Stripe   │──webhook────────────▶│ POST /ingest/:project/stripe │
│ GitHub   │──webhook────────────▶│ POST /ingest/:project/github │
│ 自定义   │──POST───────────────▶│ POST /ingest/:project/custom │
└──────────┘                      └──────────────┬───────────────┘
                                                  │ adapter.normalize()
                                                  ▼
                                            db9 events 表
```

每个 provider 一个 adapter，负责把各自格式标准化为 log9 的 event schema：

```typescript
// adapters/twilio.ts
export function normalizeTwilio(project: string, body: TwilioStatusCallback): Log9Event {
  return {
    project,
    level: body.CallStatus === 'failed' ? 'error' : 'info',
    message: `Call ${body.CallStatus}: ${body.From} → ${body.To}`,
    timestamp: new Date().toISOString(),
    tags: {
      service: 'twilio',
      callSid: body.CallSid,
      status: body.CallStatus,
      direction: body.Direction,
    },
    extra: body,  // 原始 payload 完整保留
  }
}
```

通用 custom endpoint 接受任意 JSON，做最小标准化：

```typescript
// POST /ingest/:project/custom
// Body: { level?, message, tags?, extra? }
// 适用于没有专门 adapter 的服务
```

**Webhook 验签：**

| Provider | 验证方式 |
|----------|---------|
| Twilio | `X-Twilio-Signature` + auth token HMAC |
| Stripe | `Stripe-Signature` + webhook secret |
| GitHub | `X-Hub-Signature-256` + secret HMAC |
| Agora | 签名校验 |
| Custom | Bearer token（项目级 API key） |

## 第二部分：Query Worker（统一查询接口）

Query Worker 是 log9 的**唯一查询入口**，同时服务人类和 agent。

### 两种输入模式

| 模式 | 调用方 | 输入 | 处理 |
|------|--------|------|------|
| **NL 模式** | 人类 / 浏览器 | 自然语言字符串 | Claude Haiku → SQL → 执行 |
| **Structured 模式** | Agent / API | 结构化查询对象 | 直接构建 SQL（无 LLM 调用，快且确定） |

### 查询流程

```
人类: "tuwa 最近一小时通话失败？"  ──┐
                                     ├──▶ POST /query
Agent: { project, level, since } ────┘         │
                                               ▼
                                     ┌─── body.q 存在？
                                     │
                              Yes ───┤──── No
                                │              │
                        Claude → SQL    buildStructuredQuery()
                                │              │
                                └──────┬───────┘
                                       ▼
                              校验 SELECT/WITH only
                                       │
                              Db9Gateway.query(sql)
                                       │
                              ┌────────┴────────┐
                              │                  │
                        format=json         format=html
                        (Agent/API)      (Dynamic Worker 渲染)
```

### API 定义

```typescript
// POST /query
// NL 模式
{ "q": "tuwa 最近一小时有多少 error", "format": "json" }

// Structured 模式（Agent 用，无 LLM 开销）
{
  "project": "tuwa",
  "level": ["error", "warn"],
  "since": "10m",              // 支持: 10m, 1h, 24h, 7d
  "until": "now",              // 可选
  "message_like": "%timeout%", // 可选，LIKE 匹配
  "tags": { "service": "voice-bridge" },  // 可选，JSONB 过滤
  "group_by": "message",       // 可选
  "order_by": "count",         // 可选: count, timestamp
  "limit": 20,
  "format": "json"             // json | html
}
```

### Worker 内部路由

```typescript
// routes/query.ts
export async function handleQuery(c: Context) {
  const body = await c.req.json()
  let sql: string

  if (body.q) {
    // NL 模式：Claude 生成 SQL
    const schema = await getSchema(c.env)
    sql = await generateSQL(body.q, schema, c.env.ANTHROPIC_API_KEY)
  } else {
    // Structured 模式：直接构建 SQL
    sql = buildStructuredQuery(body)
  }

  // 安全校验
  const first = sql.trim().split(/\s/)[0].toUpperCase()
  if (first !== 'SELECT' && first !== 'WITH')
    throw new Error('Only SELECT/WITH allowed')

  const result = await db9Query(c.env, sql)

  if (body.format === 'html') {
    // Dynamic Worker 渲染（m0rphic LOADER.get() 模式）
    return renderHtml(c, body.q || body, result)
  }
  return c.json(result)
}
```

### 安全层（RPC Stub 模式，复用 m0rphic）

```typescript
class Db9Gateway extends WorkerEntrypoint {
  async query(sql: string) {
    const first = sql.trim().split(/\s/)[0].toUpperCase()
    if (first !== 'SELECT' && first !== 'WITH')
      throw new Error('Only SELECT/WITH allowed')

    return fetch(`https://api.db9.ai/customer/databases/${this.env.DB9_DATABASE_ID}/sql`, {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${this.env.DB9_TOKEN}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ query: sql }),
    }).then(r => r.json())
  }
}
```

### AGENT.md

```markdown
# Log9 Agent

你是 log9 可观测性 Agent，身份为 `log9`，24/7 运行。

## 职责
持续通过 Query Worker 查询各产品日志，发现异常和优化点。

## 查询日志
不要直接连接 db9。所有查询通过 Query Worker：

# 结构化查询（推荐，快且确定，无 LLM 开销）
curl -X POST https://log9.ai/query \
  -H "X-Log9-Key: $LOG9_API_KEY" \
  -d '{"project":"tuwa","level":["error","warn"],"since":"10m"}'

# 自然语言查询（复杂分析时用）
curl -X POST https://log9.ai/query \
  -H "X-Log9-Key: $LOG9_API_KEY" \
  -d '{"q":"对比 tuwa 今天和昨天同时段的 error 率变化趋势"}'

## 每轮循环
1. `wanman recv` 查收消息
2. 通过 Query Worker 结构化查询: 最近 10 分钟各项目的 error/warn 事件
3. 需要深度分析时，用 NL 模式查询（如趋势对比、关联分析）
4. 通过 Query Worker 写入 findings（POST /ingest/log9/sdk）
5. 根据严重程度决定行动

## findings 上报
发现的问题通过 SDK ingest 端上报（findings 本身也是一种日志）：

curl -X POST https://log9.ai/ingest/log9/sdk \
  -H "X-Log9-Key: $LOG9_API_KEY" \
  -d '[{
    "level": "warn",
    "message": "error_spike: tuwa voice-bridge timeout 增加 300%",
    "tags": { "type": "finding", "severity": "high", "project": "tuwa" },
    "extra": { "evidence": [...], "status": "open" }
  }]'

## 行动决策

| 严重程度 | 行动 |
|---------|------|
| critical | `wanman send dev --steer "紧急修复: {finding}"` |
| high | `wanman send devops "发现问题: {finding}"` |
| medium | `wanman task create "优化: {title}" --assign dev` |
| low | 上报 finding，下次巡检跟踪 |

## 协作规则
- 需要修代码 → 发给 `dev`（自动 takeover repo 修复）
- 需要决策 → 发给 `ceo`
- 运维相关 → 发给 `devops`
- 需要人介入 → `wanman send human --type decision "..."`
```

### wanman runtime 自动提供的能力（log9 不用管）

- Claude Code CLI 进程管理 + 自动重启
- 消息队列（recv/send）+ steer 中断
- cron 调度（每 10 分钟触发）
- token 追踪 + 执行指标
- 与 dev/devops/ceo 等 agent 的协作通道

## 技术栈

- TypeScript 5.7 (ESM)
- Monorepo: pnpm + Turborepo
- Runtime: Cloudflare Workers (Hono)
- Database: db9.ai (Postgres)
- AI: Claude Haiku (query generation)
- Agent: wanman.ai runtime
